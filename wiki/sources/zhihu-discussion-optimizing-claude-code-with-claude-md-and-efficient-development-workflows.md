# Zhihu Discussion: Optimizing Claude Code with CLAUDE.md and Efficient Development Workflows

## Summary
A detailed Zhihu discussion thread featuring multiple, often contradictory, perspectives on configuring and using Claude Code effectively. The thread centers on the use of [[CLAUDE.md]] files for persistent memory, but expands into a broader debate about optimal AI-assisted development workflows. It includes a comprehensive configuration guide, a minimalist spec-driven approach, and practical reflections from user experience.

## Key Technical Perspectives

### Comprehensive Configuration Approach (白玉京)
The primary, detailed answer provides a structured guide to Claude Code's [[Persistent Memory System (Claude Code)]], advocating for:
- **Detailed CLAUDE.md Files**: Recommends files up to 200 lines covering project rules, architecture, coding standards, and personal preferences.
- **Permission Mode Optimization**: Explains configuring [[Permission Modes (Claude Code)]] (default, acceptEdits, plan, auto, bypassPermissions) to balance security and workflow efficiency.
- **Auto Memory Utilization**: Emphasizes leveraging Claude's ability to automatically learn and record user habits and project-specific "gotchas."
- **Rules File Integration**: Suggests creating separate `.claude/rules/` files for granular, reusable instructions.

### Minimalist, Spec-Driven Approach (yahah)
A contrasting answer from a PhD candidate advocates for a fundamentally different philosophy:
- **Extreme Brevity in CLAUDE.md**: Provides a concise template (~50 lines) focusing only on core project context and a strict prohibition against [[Vibe Coding]].
- **Workflow Over Configuration**: Shifts emphasis from internal Claude configuration to external development methodologies like [[Spec Coding vs. Vibe Coding|Spec Coding]] and the [[Humanize Workflow]].
- **Tool Chaining**: Mentions using Claude alongside other agents (e.g., Codex for review) and plugins, exemplifying [[AI Tool Chaining/Combination]].

### Pragmatic Reflection (笙囧同学)
A user shares insights from two months of trial and error, concluding that an overly long CLAUDE.md (~2000 words) reduced effectiveness. They settled on a ~800-word file that prioritizes clarity and brevity, aligning more with the minimalist philosophy.

## Core Contradictions in the Discussion

1.  **CLAUDE.md Philosophy**: A central contradiction exists between the comprehensive (白玉京) and minimalist (yahah, 笙囧同学) approaches. One advocates for exhaustive detail to guide the AI, while the other argues that concise, high-level context paired with disciplined external workflows is more effective.
2.  **Workflow Emphasis**: The discussion splits between optimizing Claude's *internal* memory and permission systems versus focusing on *external* development processes and prompt discipline. The minimalist view implies complex configuration is secondary to a robust methodology like spec-driven development.
