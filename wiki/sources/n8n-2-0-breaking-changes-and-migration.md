# n8n 2.0 - 破坏性更新与迁移指南

## 概述
n8n 2.0预览版发布，这是一次重大升级，涉及架构重构、安全策略升级、工作流行为逻辑重做，并带来多个"破坏性更新"。对于正在使用n8n的用户需要谨慎对待迁移过程，否则可能导致工作流失效。

## 关键内容

### 主要更新内容
- **UI更新**：界面优化改进
- **Publish和Activate机制变更**：发布和激活流程调整
- **子工作流更新**：子工作流调用逻辑变化

### 破坏性更新（重要）
n8n 2.0默认禁用了以下功能，直接影响现有自动化流程：
- **禁用execute command节点**：命令执行节点被禁用
- **禁止code节点读取系统环境变量**：安全策略升级
- **移除内置--tunnel隧道功能**：隧道功能不再内置

### 迁移建议
1. 先使用工作流兼容报告检查现有工作流
2. 使用官方迁移工具进行迁移
3. 在测试环境验证后再升级生产环境
4. 注意破坏性更新对现有流程的影响
