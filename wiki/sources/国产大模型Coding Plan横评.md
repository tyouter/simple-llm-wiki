# 国产大模型横评：Coding Plan避坑指南

## 测试背景

2026年初，国产大模型集中发布更新，野码AI爆肝5小时实测GLM-5、MiniMax-M2.5、Qwen3.5-plus在复杂项目开发中的表现。

## 测试方法

### 开发环境
- 在VS Code中进行开发
- 使用web工具并行测试多个模型
- 建立docs目录放置开发文档

### 测试维度
1. UI原型设计能力
2. Skills技能调用能力
3. 项目开发完成度
4. Tokens消耗成本

## 测试结果

### GLM-5
- Pro版本测试，tokens消耗了100%
- 只输出了23%进度，连完整UI原型都没看到
- 购买套餐成本高昂
- 需要通过其他平台完成项目

### MiniMax-M2.5
- 开发速度最快
- UI效果和还原度最高
- Tokens仅消耗套餐额度的10%
- 性价比最高

### Qwen3.5-plus
- 使用Claude Code的Cos模式
- 基本完成开发
- 工具调用与MiniMax原型方法相似
- 实际费用约30元以上

## 关键发现

1. 不同模型的Coding Plan表现差异巨大
2. MiniMax-M2.5在性价比和效率上表现最佳
3. 部分模型需要额外配置才能正常运行
