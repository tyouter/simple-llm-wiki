# Claude Code开发者踩坑经验分享

## 来源背景

Claude Code开发工程师发布了两篇构建Claude Code时的经验分享。
适用于：
- Claude Code使用者：提升工作效率
- 智能体工具开发者：理解设计理念

## 核心设计原则

### 工具设计理念
- 让工具能够调用一些工具
- 工具设计要符合人类思维习惯
- 智能体工具构建是一门艺术，不是技术

### 结构化输出
- 让模型输出结构化内容（如Claude格式）
- 用户进行回答
- 发现结构化输出不完全按预期执行时容易出错

## 关键发现

### ask_user工具
- 发现让Claude在任何时候都可以调用ask_user工具
- 符合人类的思维和习惯
- 任务执行时应该使用习惯性的交互方式

### 计划执行问题
- 有了计划后可能直接跳步骤执行
- 不按计划一步步来
- 需要live_output/live_task理念来解决

### 上下文获取
- RAG检索方式在各种环境下不稳定
- 模型自己去上下文比自动检索效果更好
- 让模型自己构建上下文，而非强行塞入

## 渐进式披露概念

Skills引入后正式提出渐进式披露：
- Skills可以让模型去看相关内容
- 完成任务的方式更符合人类习惯
- 模型能力在不断进步，可能不再需要检索方式

## 工具添加原则

- 每个工具都要精细考虑是否应该加
- 不是所有工具都需要一次性加载
- 模型需要时自己去调用
- 这样提示词就不会过载上下文

## 核心洞见

- 智能体构建是一门艺术
- 工具要符合人类逻辑和思维
- 不能想当然把智能体当做工具，要当做有思维的协作伙伴
