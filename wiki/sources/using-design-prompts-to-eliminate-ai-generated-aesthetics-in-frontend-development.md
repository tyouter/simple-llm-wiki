This document by [[Rrupmid Nyche]] presents a method for generating high-quality web interfaces without the recognizable "AI aesthetic" using specialized design prompts from [[designprompts.dev]]. The approach combines [[Prompt Engineering for Aesthetic Control]] with [[AI Tool Chaining/Combination]] to move beyond generic [[Tailwind]] styles and Claude's default visual tendencies ("Claude味").

## Core Method
1. **Curate Design Prompts**: Download XML prompt files from [[designprompts.dev]], a repository of previewable style prompts that represent the high end of the [[AI-Generated Design Quality Spectrum]].
2. **Guide Generation**: Feed these prompts to [[Claude Code]] to generate initial interface code, explicitly instructing it to avoid AI-looking aesthetics.
3. **Optional Enhancement**: For superior results, pass the generated code to tools identified as [[AI Tool Specialization for Design|specialized for frontend design]], such as [[Gemini]] or [[Antigravity]].

## Advanced Techniques
- **Design System Synthesis**: Ask [[Claude Code]] to analyze existing app code and reference prompts to generate a unified `MyAppDesign.xml` file, demonstrating [[Design System Synthesis by AI]].
- **Rapid Exploration**: This [[Vibe Coding]] approach enables quick generation of "画风各异的 landing page" (landing pages with different artistic styles) by swapping design prompts.

The workflow exemplifies an [[AI-Native Design Workflow]] where human [[Aesthetic Judgment in Tech]] directs AI agents through curated constraints, achieving [[AI-Free Interface Aesthetics]] through strategic tool use and prompt management.
