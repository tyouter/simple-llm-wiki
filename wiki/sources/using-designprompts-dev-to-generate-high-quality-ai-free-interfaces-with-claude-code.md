# Using DesignPrompts.dev to Generate High-Quality AI-Free Interfaces with Claude Code

## Summary
This workflow, documented by [[Rrupmid Nyche]], demonstrates how to use the curated prompt repository [[designprompts.dev]] to guide [[Claude Code]] in generating web interfaces that avoid stereotypical "AI aesthetics." The method focuses on producing unique, high-quality frontend code by leveraging pre-built XML design prompts.

## Core Workflow
1.  **Browse and Select Prompts:** Visit [[designprompts.dev]] to browse its collection of over 20 previewable design prompts in various styles (e.g., Brutalist, Retro, Minimalist).
2.  **Download and Combine:** Download multiple XML prompts that match the desired aesthetic. These prompts can be combined or used as references.
3.  **Adapt for Your Project:** Synthesize a new, custom design specification (e.g., `MyAppDesign.xml`) tailored to your specific application by instructing Claude Code to merge elements from the downloaded prompts.
4.  **Generate Code:** Provide the final, adapted XML prompt to [[Claude Code]] to generate the corresponding JSX and CSS code.
5.  **Optional Tool Chaining:** For potentially superior results, especially in frontend design, consider using the generated output as a starting point for other specialized AI tools like [[Gemini]] or [[Antigravity]].

## Key Principles
- **Avoiding AI Clichés:** The primary goal is to escape recognizable AI visual tropes, such as generic gradients or overused [[Tailwind CSS]] patterns often associated with "Claude味" (Claude flavor).
- **Leveraging Curation:** Relies on the pre-curated, high-quality prompts from [[designprompts.dev]] rather than crafting detailed prompts from scratch, exemplifying the practice of [[Design Prompt Curation/Repositories]].
- **Synthesis Over Direct Copy:** The process involves creative adaptation and combination of prompts, a technique known as [[Prompt Combination & Adaptation]], to create a bespoke design direction.
