# Decomposition as Core Skill

## Definition
In an AI-assisted software development workflow, the primary technical skill shifts from writing correct and elegant syntax to the ability to break down a complex, high-level goal or problem into a series of independent, clear, and parallelizable tasks that can be effectively executed by AI agents (like [[Claude Code]]).

## Why It Becomes Central
With the **[[Commoditization of Execution]]** of code by AI, the bottleneck is no longer *how* to implement a function, but *how to describe what needs to be implemented* in a way that an AI can act upon reliably and efficiently. This requires a deep understanding of system architecture, dependencies, and interfaces.

## Practical Application
As described in the **[[Zhihu Discussion: Claude Code's Ecosystem, Impact on Engineering, and Practical Experiences]]**, this skill is essential for leveraging tools like **[[Clave]]** or workflows like **[[GStack: Role-Based AI Development Workflow for Claude Code]]**, which are designed for parallel execution. The engineer must decompose a project into modules that can be assigned to separate AI agents or sessions working simultaneously.

## Connection to Other Concepts
- **[[Engineer Identity Transformation]]**: Decomposition is a key activity in the new engineer's role as an orchestrator.
- **[[Agent Operating System]]**: Managing a multi-agent system requires clean task decomposition to avoid conflicts and ensure integration.
- **[[GStack: Role-Based AI Development Workflow for Claude Code]]**: The `/plan-engineer` command is essentially a decomposition prompt, translating a vision into a technical plan.

## Developing the Skill
This involves thinking in terms of APIs, contracts, and discrete units of work rather than continuous lines of code. It emphasizes planning and design thinking before any AI is prompted.
