# Vibe Coding Enhancement Workflow

## Definition
A structured, two-step method for improving the aesthetic output of [[Vibe Coding]]. It addresses the common pitfall of generic, "AI-looking" interfaces by layering a foundational design system prompt with a specific, curated style prompt.

## The Two-Step Process

1.  **Foundation Layer (System Prompt):**
    - **Purpose:** To override the AI assistant's (e.g., [[Claude]]) default aesthetic biases and establish core design principles.
    - **Content:** A comprehensive [[Design Style System Prompt]] that covers typography, color theory, animation guidelines, layout logic, and background design rules.
    - **Outcome:** Shifts the baseline output from a generic 50/100 quality score to a more competent, principled starting point (~60/100).

2.  **Style Injection Layer (Curated Prompt):**
    - **Purpose:** To apply a distinct, refined visual aesthetic that feels intentional and human-designed.
    - **Content:** A pre-engineered prompt copied from a [[Curated Design Prompt Repositories]] like [[designprompts.dev]]. This prompt encapsulates a specific style (e.g., "Glassmorphism," "Retro Futurism").
    - **Outcome:** Elevates the design to a professionally competent level (~80/100), achieving [[AI-Free Interface Aesthetics]].

## Connection to Broader Concepts
- This workflow is a practical application of [[Prompt Combination & Adaptation]].
- It operationalizes the theory behind platforms like [[designprompts.dev]] for direct use with [[Claude]].
- It exemplifies how to navigate the [[AI-Generated Design Quality Spectrum]] intentionally.

## Why It Works
By separating foundational design rules from stylistic flavor, this workflow gives the AI clear, non-conflicting directives. The system prompt sets the "rules of the game," while the curated prompt provides the "visual theme." This combination is more effective than a single, overly complex prompt and produces more consistent results.
