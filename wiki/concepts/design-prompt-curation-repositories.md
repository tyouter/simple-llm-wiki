# Design Prompt Curation/Repositories

## Definition
The practice of collecting, maintaining, and sharing pre-built, high-quality prompts specifically engineered to guide AI tools toward generating desirable and specific design outputs, particularly for user interfaces and frontend code.

## Purpose
These repositories address the challenge of prompt engineering for design by providing reliable, tested starting points. They help users bypass generic AI outputs and achieve more targeted, professional, or stylistically distinct results without needing to be expert prompt engineers themselves.

## Key Characteristics
- **Pre-Built & Previewable:** Prompts are often complete, ready-to-use XML or structured text blocks, accompanied by visual previews of the expected output style.
- **Categorized by Style:** Collections are typically organized by aesthetic (e.g., Brutalist, Retro, Minimalist, Glassmorphism).
- **Focused on Implementation:** The prompts are designed to generate actual code (like JSX and CSS) rather than just descriptive text or images.

## Example: [[designprompts.dev]]
A niche website that exemplifies this concept. It offers a curated collection of over 20 design prompts, allowing users to browse styles, preview the aesthetic, and download the corresponding XML prompt to use with AI coding assistants like [[Claude Code]].

## Value in Workflows
Using such a repository is a core step in workflows aimed at achieving [[AI-Free Interface Aesthetics]]. It provides the raw material for techniques like [[Prompt Combination & Adaptation]] and enables [[Vibe Coding]] by offering concrete "vibes" in prompt form.
