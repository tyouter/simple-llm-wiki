# AI-Native Design Workflow

## Definition
A design workflow centered on AI generation and iteration, compressing the cycle from idea to clickable prototype to minutes. It involves a structured process of prompt input → AI generation → preview/iteration → code export, potentially reshaping traditional UI/UX processes.

## Core Process
1. **Prompt Input:** User provides natural language description of design intent
2. **AI Generation:** AI (like [[Google Stitch]]) generates complete interface designs
3. **Preview/Iteration:** User reviews and iterates through refined prompts
4. **Code Export:** Final designs can be exported as production-ready code

## Comparison to Traditional Workflow
- **Traditional:** Sketch → Wireframe → Mockup → Prototype → Handoff (days/weeks)
- **AI-Native:** Prompt → Generated Design → Iteration → Export (minutes/hours)

## Tools and Platforms
[[Google Stitch]] represents a pure implementation of this workflow, offering an end-to-end AI-native design environment. Other tools may implement aspects of this workflow within traditional design tools.

## Implications for Designers
- **Skill Shift:** Emphasis moves from manual design skills to [[Design Prompt Engineering for UI]]
- **Speed Increase:** Dramatic compression of design iteration cycles
- **Accessibility:** Lower barrier to entry for basic interface creation
- **Quality Concerns:** Risk of [[Prompt-Dependent Output Quality]] and generic [[AI-Free Interface Aesthetics|AI aesthetics]]

## Relationship to Other Concepts
