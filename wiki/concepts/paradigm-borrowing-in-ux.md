**Paradigm Borrowing in UX** involves adopting successful interaction patterns, design metaphors, or workflow models from one software application and implementing them in another. This cross-pollination accelerates UX innovation by leveraging proven solutions rather than reinventing interfaces from scratch.

The [[Obsidian Dragger]] plugin explicitly engages in paradigm borrowing by bringing "[[Notion]]-like" drag-and-drop functionality to Obsidian. This recognizes that Notion's block-based manipulation has set user expectations for fluid content rearrangement. The borrowing isn't slavish imitation but thoughtful adaptation to Obsidian's markdown-based, file-centric context.

This practice relates to [[Aesthetic Judgment in Tech]], where designers and developers make conscious choices about which borrowed paradigms align with their application's philosophy and user needs. Successful paradigm borrowing requires understanding not just the surface interaction but the underlying mental model—in this case, treating content as discrete [[Text Block as a Unit of Manipulation|blocks]] rather than continuous text. When done well, it enhances applications without diluting their core identity.
