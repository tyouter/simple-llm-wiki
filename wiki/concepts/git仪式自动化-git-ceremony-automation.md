**Git仪式自动化**指的是将软件开发中日常的、重复性的Git版本控制操作，封装为可一键执行或自动触发的AI技能。这些操作通常包括同步主分支、运行测试、提交代码、推送更改、创建拉取请求（PR）以及更新PR描述等。

在[[GStack]]中，`/ship`技能专门负责处理这些“Git仪式”。虽然每个单独的操作（如`git pull --rebase`）都很简单，但开发者每天可能手动执行十几次，累积起来消耗显著的认知资源和时间，并容易因疏忽出错。自动化这些仪式减少了上下文切换，降低了人为错误风险，并确保了团队流程的一致性。

这是[[AI Tool Specialization]]在提升开发效率“最后一公里”上的经典应用。它解放了开发者，让他们更专注于创造性的编程和问题解决工作。在[[基于角色的AI开发 (Role-Based AI Development)]]范式中，`/ship`技能扮演了“发布经理”或“集成工程师”的角色，负责将代码安全、规范地送入协作流程。它也是[[Agent Skills Framework]]中一个看似简单但实用性极高的技能范例，展示了如何通过AI将最佳实践固化为无缝的体验。
