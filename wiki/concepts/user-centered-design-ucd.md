# User-Centered Design (UCD)

## Definition
A broad design philosophy and process framework that places the needs, wants, and limitations of end users at the center of every stage of the design process. The ultimate goal is to create products that are highly usable and provide a positive user experience.

## Core Principles
1.  **Early and Continuous Focus on Users**: Understanding users is not a one-time research phase but an ongoing activity.
2.  **Empirical Measurement**: Using usability testing and observation with real users to guide design decisions, not just opinion or precedent.
3.  **Iterative Design**: The process is cyclical—design, test, measure, and refine—acknowledging that the first design is rarely the best.

## UCD Toolkit
As highlighted by [[郑坚义]] in the [[Zhihu Discussion: The Role of 'Scenarios' in Product Management]], UCD employs a suite of complementary tools for requirement analysis and design. Key interrelated tools include:
- **[[Persona (用户画像)]]**: A semi-fictional archetype representing a key user segment. Answers "*Who* are we designing for?"
- **[[Scenario Analysis (场景分析)]]**: A narrative description of a user achieving a goal in a specific context. Answers "*When, where, and why* would they use our product?" and "*What* is their story?"
- **[[Use Case (用例)]]**: A detailed, step-by-step description of the interactions between a user (actor) and the system to accomplish a specific task. Answers "*How* do they use it?"

These tools form a hierarchy: Personas define the users, Scenarios define the contexts and goals, and Use Cases define the specific interactions within those scenarios.

## Relationship to Scenario Analysis
[[Scenario Analysis (场景分析)]] is not a standalone activity but a critical component of the UCD process. It provides the rich, contextual "story" that bridges the abstract understanding of a user (Persona) and the concrete specification of system behavior (Use Case). It ensures that features are grounded in real human contexts rather than theoretical functionality.
