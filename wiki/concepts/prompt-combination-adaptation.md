# Prompt Combination & Adaptation

## Definition
A prompt engineering technique where multiple existing prompts are merged, synthesized, or used as references to create a new, custom prompt tailored to a specific project's needs. In design contexts, this often involves blending stylistic elements from different sources.

## Process in AI-Assisted Design
1.  **Gathering Sources:** Downloading multiple design prompts from a repository like [[designprompts.dev]]. For example, one prompt for a "Brutalist" layout and another for a "Retro" color scheme.
2.  **Analysis and Synthesis:** Instructing the AI (e.g., [[Claude Code]]) to analyze the provided prompts and extract their core principles, components, or stylistic rules.
3.  **Creation of a New Specification:** Directing the AI to combine these elements into a single, coherent new design specification document (e.g., `MyAppDesign.xml`) that fits the requirements of an existing application.
4.  **Iteration:** The new, adapted prompt is then used to generate the final code.

## Benefits
- **Uniqueness:** Creates outputs that are not direct copies of any single source prompt, helping achieve [[AI-Free Interface Aesthetics]].
- **Specificity:** Tailors the design direction precisely to the project context, rather than applying a generic template.
- **Efficiency:** Leverages the hard work of prompt curation while allowing for creative customization.
