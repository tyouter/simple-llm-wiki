# Design Style System Prompt

## Definition
A comprehensive, foundational prompt used in [[AI-assisted design]] to establish core aesthetic principles and override an AI model's default stylistic preferences. It acts as a "design constitution" for the generation session.

## Purpose
Raw [[Vibe Coding]] often fails because AI models like [[Claude]] have ingrained, generic visual defaults. A Design Style System Prompt directly counteracts this by providing explicit, high-level rules before any specific content or style is requested. Its goal is to raise the baseline quality of all subsequent AI-generated output.

## Typical Components
A robust system prompt will include directives on:
- **Typography:** Rules for font pairing, hierarchy, sizing, weight, and line spacing.
- **Color:** Guidance on palette construction, contrast ratios, and color application (primary, secondary, accent).
- **Animation & Interaction:** Principles for motion (e.g., "smooth and purposeful"), hover effects, and transitions.
- **Layout & Spacing:** Grid systems, use of white space, and compositional balance.
- **Background & Depth:** Treatment of backgrounds, layers, and visual depth (avoiding flat, monotonous layouts).

## Role in the Workflow
In the [[Vibe Coding Enhancement Workflow]], the Design Style System Prompt is the essential first step. It is applied at the system level (often in a dedicated "system" instruction field) to set the stage. Once these foundational rules are in place, a second, more specific style prompt (from a [[Curated Design Prompt Repositories]]) can be layered on top to apply a particular aesthetic (e.g., "Cyberpunk," "Minimalist") without conflicting with core design logic.

## Key Insight
This concept acknowledges that AI needs explicit design education. It's not enough to ask for a "beautiful website"; you must first teach the AI what your fundamental principles of beauty are.
