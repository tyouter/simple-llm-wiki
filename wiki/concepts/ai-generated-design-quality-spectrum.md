# AI-Generated Design Quality Spectrum

## Definition
A conceptual framework for categorizing the aesthetic output of AI design tools (like [[Claude]] used for [[Vibe Coding]]) on a qualitative scale, recognizing that results are not binary (good/bad) but exist on a continuum.

## The Spectrum
Based on analysis, outputs can be roughly scored:

- **~50/100 (Generic / "AI-Looking"):** The default output of raw, unguided vibe coding. Functional but visually bland, using common tropes from the AI's training data. Easily identifiable as machine-generated.
- **~60/100 (Passable / Competent):** Achieved by applying foundational guidance, such as a [[Design Style System Prompt]]. The design has basic principles (good spacing, typography) but may lack a distinct or refined style.
- **~80/100 (Professional / Intentional):** The target of enhanced workflows. Achieved by combining a system prompt with a specific style from a [[Curated Design Prompt Repositories]]. The output feels deliberately designed, exhibits a coherent aesthetic, and approaches [[AI-Free Interface Aesthetics]].

## Utility of the Concept
This spectrum helps set realistic expectations and provides a roadmap for improvement. It:
1.  **Acknowledges a Baseline Problem:** It contradicts the marketing hype that AI produces perfect design instantly, confirming that raw outputs are often subpar.
2.  **Provides a Progress Metric:** Designers and developers can assess where their current outputs fall and what techniques (e.g., implementing the [[Vibe Coding Enhancement Workflow]]) might move them up the spectrum.
3.  **Justifies Prompt Engineering:** It creates a clear value proposition for using structured prompts and curated resources—they are the tools needed to climb from 50 to 80.

## Implication
The existence of this spectrum suggests that design expertise remains valuable. The prompts that generate 80/100 results are themselves products of design knowledge, either from the user or the curator of a prompt repository.
