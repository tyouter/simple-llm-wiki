# Clave

## Description
A desktop application built by the developer "穆双" (Mu Shuang) using [[Claude Code]]. Clave is designed to manage multiple parallel Claude Code sessions through a kanban board interface. It serves as a practical tool for implementing an **[[Agent Operating System]]**-like workflow, where different AI agents can work on separate, decomposed tasks simultaneously.

## Function and Purpose
- **Parallel Session Management**: Allows users to create, view, and manage multiple independent Claude Code workspaces (sessions) side-by-side.
- **Kanban Board Visualization**: Uses a board with columns (e.g., Todo, In Progress, Done) to track the status of different development tasks being handled by AI agents.
- **Workflow Enabler**: Facilitates the **[[Decomposition as Core Skill]]** by providing a visual tool to assign decomposed tasks to different AI sessions, aligning with principles of parallel execution found in **[[GStack: Role-Based AI Development Workflow for Claude Code]]**.

## Significance
Clave is cited in the **[[Zhihu Discussion: Claude Code's Ecosystem, Impact on Engineering, and Practical Experiences]]** as a concrete example of the new AI-assisted engineering paradigm. It is both a product *of* the new workflow (built with Claude Code) and a tool *for* that workflow (enabling parallel agent management).

- **Ecosystem Example**: It is a prominent tool listed in the **[[awesome-claude-code]]** repository.
- **Identity Transformation Tool**: It operationalizes the shift from a single-coder mindset to an orchestrator mindset central to **[[Engineer Identity Transformation]]**.

## Connection to Concepts
Clave directly implements the idea of running multiple AI agents in parallel, a core tenet of an **[[Agent Operating System]]**. It provides the interface for engineers to practice **[[Decomposition as Core Skill]]** and manage the **[[Commoditization of Execution]]** provided by multiple Claude Code instances.
