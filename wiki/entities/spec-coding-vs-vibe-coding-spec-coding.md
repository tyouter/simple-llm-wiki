# Spec Coding（规范编码）

## 概述
规范编码是一种AI辅助开发的编码方法论，强调通过明确规格和需求文档引导AI生成代码，与意念编码(Vibe Coding)形成对立，主张需求驱动开发而非依赖模糊意图或即兴指令。

## 基本信息
- 类型：编码方法论
- 相关领域：AI辅助开发、需求工程、规范驱动开发

## 核心特征

### 规格优先
- 先定义明确规格再编码
- 需求文档驱动AI生成
- 禁止依赖模糊意图

### 与意念编码对比
| 维度 | 规范编码 | 意念编码 |
|------|---------|---------|
| 输入 | 明确规格 | 模糊意图 |
| 过程 | 需求驱动 | 即兴指令 |
| 结果 | 可预测 | 不可控 |

## 在Claude Code中的体现
知乎讨论中yahah的观点体现规范编码思想：
- CLAUDE.md极简，约50行聚焦核心上下文
- 强调外部开发方法论而非内部配置
- 禁止[[意念编码]]，强调需求驱动

## 实践原则
- 先规划后编码（Plan Mode）
- 明确约束比列举允许更有效
- "不要"比"要"更重要
