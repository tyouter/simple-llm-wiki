# Obsidian

## Overview
A knowledge base and note-taking software application that operates on locally stored Markdown files, featuring linking, backlinking, and a rich plugin ecosystem. Known for its flexibility and focus on creating personal knowledge graphs.

## Core Features

### File-Based Architecture
- **Local Storage**: All notes stored as plain Markdown files on the user's device.
- **Future-Proof Format**: Markdown ensures notes remain accessible even without Obsidian.
- **File System Integration**: Works directly with folder structures on disk.

### Knowledge Graph Capabilities
- **Bi-Directional Linking**: Create connections between notes with automatic backlinks.
- **Graph View**: Visual representation of note relationships.
- **Unlinked Mentions**: Discover potential connections between notes.

### Extensibility
- **Plugin Ecosystem**: Community-developed plugins extend functionality significantly.
- **Custom CSS**: Visual customization through style snippets.
- **API Access**: Developers can create integrated tools and workflows.

## Common Use Cases
- **Personal Knowledge Management**: Building interconnected notes on various topics.
- **Research and Writing**: Organizing sources, ideas, and drafts.
- **Project Management**: Task tracking, meeting notes, and documentation.
- **Learning Journals**: Tracking progress and insights in skill development.

## Integration with AI and Automation
- **[[AI-Enhanced Note-Taking]]**: AI agents can generate and structure Obsidian notes.
- **[[obsidian-skills]] Toolkit**: Official toolkit enabling AI to create native Obsidian content.
- **Workflow Automation**: Combining with other tools through scripts and APIs.

## Version Control and Synchronization
- **[[Git-Based Knowledge Management]]**: Many users manage vaults with Git for version control.
- **Sync Services**: Official sync service and third-party options (Dropbox, iCloud, etc.).
- **Mobile Access**: Companion apps for iOS and Android.

## Community and Philosophy
- **User-Driven Development**: Strong community influence on feature development.
- **Privacy Focus**: Local-first approach appeals to privacy-conscious users.
- **Minimalist Aesthetic**: Clean interface that stays out of the way of content.

## Related Concepts
- **[[Note Organization Philosophy]]**: Obsidian supports multiple organizational approaches.
- **[[Dataview]]**: A popular plugin for querying and displaying note data.
- **[[Markdown as Human-AI Interface]]**: Obsidian's use of Markdown facilitates AI collaboration.

**Obsidian** is a knowledge management and note-taking application built on local markdown files, renowned for its extensibility through community plugins. Unlike cloud-based alternatives like [[Notion]], Obsidian emphasizes user ownership, privacy, and network thinking through its signature graph view of interconnected notes.

The application's minimalist core design intentionally leaves room for extension via its vibrant [[Obsidian Plugin Ecosystem]]. This ecosystem includes hundreds of community-developed plugins that add functionality ranging from advanced queries (Dataview) to automation (Templater) to UX enhancements like [[Obsidian Dragger]]. This extensibility allows Obsidian to compete with more opinionated, monolithic applications while maintaining its philosophical commitment to plain text and local storage.

Obsidian's architecture supports [[AI-Enhanced Note-Taking]] through plugins that integrate AI capabilities, and its file-based approach enables sophisticated [[AI Tool Chaining/Combination]] with external tools. The application has cultivated a strong culture of [[Community-Driven Feature Development]], where users frequently become developers to address their workflow needs, as seen in the creation of plugins like Obsidian Dragger by [[Ariestar]].
