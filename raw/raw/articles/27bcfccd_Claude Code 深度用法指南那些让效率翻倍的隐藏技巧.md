在社交媒体上介绍后，帖子迅速获得超过两百万次曝光。光这个数字，就说明这个痛点有多普遍。 解决什么问题？ 让 Claude 执行一个耗时任务时，你突然想顺口问一句无关的问题，比如”那个配置文件放在哪个目录来着”。一旦问出去，Claude 会停下来回答，上下文里多了一段和任务完全无关的对话，后续执行很可能就偏了。 这叫上下文污染，深度使用者几乎都踩过这个坑。 有了 /btw，你可以在任务进行期间插入一个旁路问题。 打 /btw，空格，写你的问题，发送。 Claude 会在一个独立进程里回答你，与主任务完全并行。看完答案，按空格或回车关闭这段对话，它就消失了——不进入对话历史，不影响主线任务，上下文保持干净。 额外好处：因为复用了当前提示缓存，几乎不消耗额外 token。 用过就回不去，这句话不夸张。 02 · /rewind — 代码与对话可分别独立回退 按两下 Esc 就能触发，也可以直接打 /rewind。功能类似设计软件里的 Ctrl+Z，但比那个强得多。 这个命令早就存在，但今年 2 月做了一次关键升级：代码和对话现在可以分开回退。 以前的逻辑是全量撤销，回退一次，讨论记录和代码改动全部一起没了。 现在打 /rewind 会弹出一个选择菜单，可以从历史节点中选择： • 代码 + 对话一起回退 • 只回退对话，保留代码 • 只回退代码，保留对话 • 从该节点压缩对话，释放上下文空间 最实用的场景： 让 Claude 试一种新的实现思路，跑完发现效果不理想——这时只回退代码、保留对话。Claude 还记得整个讨论脉络，知道这条路走不通，可以直接换方向，不需要重新铺垫背景。 以前遇到这种情况，只能硬着头皮 git reset，搞不好还搞乱。现在 /rewind 一键解决。 03 · /insights — 让 Claude Code 反过来分析你的用法 这个命令被严重低估了。 /insights 会生成一份本地 HTML 报告，分析你过去一个月的使用习惯：最常触发哪些命令、哪些操作存在重复模式、哪里有优化空间，并据此推荐适合你的自定义命令和 Skills。 本质上，是 Claude Code 在观察你怎么用它。 直接打 /insights，等几秒，一份格式完整的分析报告就生成在本地了。 内容很详细，不只是数据统计，还有具体的使用建议，甚至会把你某次奇怪操作的”翻车现场”记录下来。 这种细节它都记着—— 建议每个月跑一次。不是为了优化效率，而是让你重新审视自己的操作习惯，往往会有意外收获。 04 · /model opusplan — Pro 用户的省 token 神器 这是一个真正意义上的隐藏命令，在 /model 的常规切换列表里找不到它。 启用方式：直接输入 /model opusplan。 它的逻辑是：规划阶段用 Opus 4.6 深度推理，执行阶段自动切换到 Sonnet 4.6。 为什么这对 Pro 用户很重要？ 20 美元的 Pro 订阅，Opus 的可用额度非常有限。如果全程用 Opus 写代码，可能任务做到一半就开始限速。 但仔细想想，规划和写代码对模型的要求本来就不一样： • 规划需要理解整体架构、梳理模块依赖、做出判断——这件事 Opus 确实比 Sonnet 强 • 写代码对于中小型项目，Sonnet 完全够用，速度还快很多 /model opusplan 正好把两者分开，用刀刃的地方用 Opus，其余交给 Sonnet，把有限的额度花在最值得的地方。 05 · /simplify — 三个 Agent 同时帮你审代码 这是今年 2 月底集成进来的内置 Skill，核心团队成员 Boris 说他自己每天都在用。 运行 /simplify 后，Claude Code 会同时启动三个并行 Agent，分别从三个维度审查你最近的代码改动： 代码复用 — 有没有重复造轮子 代码质量 — 有没有可以简化的写法 运行效率 — 有没有性能上的浪费 三路并行跑完，汇总结果告诉你哪里可以优化。 AI 生成的代码有个通病：冗余。多余的 import、重复的逻辑片段、可以用更简洁写法替代的地方。这些细节人工 review 容易漏，/simplify 基本都能挑出来。 我的习惯是：每完成一个阶段性的功能迭代，跑一遍 /simplify。相当于三个同事在同时帮你做代码审查。 06 · /branch — 在当前进度上开出平行宇宙 这个命令之前叫 /fork，现在改名为 /branch，旧命令还能用，会自动跳转过去。 功能就是一句话：把当前对话复制一份，开出一条新会话，原有的不受影响。 典型使用场景： Claude 刚帮你梳理完一个方案的整体思路，你想试两种不同的实现路径，又不想丢掉当前的讨论积累——/branch 一下，两条线各自跑，最后对比结果选最好的。 跟 /rewind 的区别很好理解： /rewind 是后悔药，回到过去 /branch 是平行宇宙，同时往两个未来走 07 · /loop — 定时循环执行，不用人盯 /loop 让 Claude 按照你设定的时间间隔，反复执行同一个任务。 用法：/loop [间隔时间] [任务描述] 比如 /loop 5m 检查部署状态 ，就是每五分钟自动检查一次，不用你守在旁边。默认间隔是 10 分钟 最大的优势是：执行结果直接留在对话上下文里，Claude 可以根据每次的结果做出判断和后续操作，不是单纯的机械重复。 一个值得注意的细节：定期任务在创建后 3 天自动过期。任务会最后触发一次，然后自我删除。这个限制防止被遗忘的循环无限运行下去。 如果需要长期运行，官方建议使用桌面版。 08 · /remote-control — 用手机远程操控你的终端 这个功能今年 2 月底上线，简称 /rc，也可以打完整的 /remote-control。 运行后，它会生成一个 URL。 用手机打开这个链接，你在终端里的整个 Claude Code 会话就映射到手机屏幕上了。 两端完全同步——手机发指令，终端看得到；终端有输出，手机实时刷新。你可以在两个设备上交替操作，对话历史完全一致。 安全性没有折扣： 代码始终在本机运行，文件系统、MCP 服务器、项目配置全部留在本地。手机只是一个操作窗口，不持有任何数据。 离开电脑又想继续推进任务时，这个功能很实用。 09 · /export — 把整段讨论导出成 Markdown 用法极简：打 /export，当前会话的全部内容就导出为一个 Markdown 文件。 这个功能的价值在于保留思考过程。 你和 Claude 围绕一个架构方案讨论了半小时，中间有大量的权衡、否定、转折和最终结论。这些推导过程如果不保存，下次再面对同一个问题，等于从零开始。导出成文件，后续可以直接作为新会话的上下文喂进去。 另一个用法：把对话导出后交给其他工具（比如 Codex）做交叉验证——让不同的 AI 互相审查，有时候能发现单一视角下容易忽略的问题。 10 · 快捷键备忘 最后整理几个容易被忽略但很实用的快捷键： Ctrl+V — 直接粘贴截图 不需要先保存成文件再上传。遇到报错截个图，Ctrl+V 直接粘进去，Claude 看图分析。 ⚠️ Mac 用户注意：这里是 Ctrl+V，不是 Cmd+V 2 Ctrl+J / Control+J — 输入框内换行 3 Ctrl+R — 搜索历史 prompt 4 Ctrl+U — 清空当前输入行 结语 写到这里，有一个感受越来越强烈：Claude Code 的进化速度，已经到了一种让人来不及消化的程度。 上面这些功能，有的是三月刚出的，有的是二月底才集成进来的。版本日志里记录的功能已经够多了，还有些连日志都没写，只在开发者的社交媒体上出现过一句。 追不上所有更新是正常的，但保持一定的关注度是值得的。官方的变更日志在这里持续更新：
