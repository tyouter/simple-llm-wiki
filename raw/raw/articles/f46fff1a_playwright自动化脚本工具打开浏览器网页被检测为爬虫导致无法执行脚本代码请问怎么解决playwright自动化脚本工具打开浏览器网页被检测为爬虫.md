一跑就被知乎拦了。弹了个验证码，说我网络环境异常。 那会儿我还纳闷，就抓自己的页面也拦？去Console里查了一圈才搞明白——Playwright默认启的Chromium跟裸奔没区别。navigator.webdriver直接返回true，相当于脸上写着”我是脚本”。User-Agent带HeadlessChrome，插件列表是空的，WebGL信息也不对。知乎JS检查这几样东西就够判断了。 这一层倒好解决。pip装个playwright-stealth就行，2.0.2版本，今年2月刚更新。它的活就是往页面注入JS把这些暴露点盖掉。另外浏览器启动的时候加一句--disable-blink-features=AutomationControlled，从Blink引擎层面就不让webdriver标志亮出来了。 配完是这样： BROWSER_ARGS = ["--disable-blink-features=AutomationControlled", "--no-sandbox", "--disable-dev-shm-usage"] stealth_config = StealthConfig( webdriver=True, webgl_vendor=True, navigator_plugins=True, navigator_languages=True, navigator_user_agent=True, chrome_runtime=True, languages=("zh-CN", "zh", "en"), vendor="Google Inc.", renderer="ANGLE (Intel, Intel(R) UHD Graphics 630, OpenGL 4.5)", ) renderer那行写Intel UHD Graphics 630，就是最普通的集显，到处都是。不改的话可能暴露你在虚拟环境跑。 加上这些重新来，验证码消失了。25分钟搞定。 然后掉进了第二个坑。这坑比第一个难受多了，因为它看起来根本不像是个”坑”。 知乎有些指标要登录才显示，比如浏览量。我就打开登录页准备扫码。结果二维码那块就是一个灰色空框，什么也没有。切密码登录，按钮点了跟没点一样。等了半天，页面其他部分都正常，就登录交互这块废了。 这事跟反爬一点关系没有。 Playwright带的Chromium不是完整的Chrome。它阉割了一些东西，知乎登录页刚好用到了这些被阉割的部分。stealth管的是”让网站别发现你是机器人”，管不了”浏览器本身缺胳膊少腿”。 我在这上面卡了35分钟。比第一层多十分钟。主要是表现太有迷惑性——页面能打开不报错，你会以为是stealth没配对或者哪个参数没加，来回试。 其实看清这是Chromium环境问题之后，解路就很窄了。我翻了下这个问题赞最高的回答，测试逆转大师那个13赞的，列了三种方案：用channel参数启动本地Chrome、加载本地用户缓存、CDP远程调试。都能绕过去。但CDP每次得先手动开Chrome带调试端口太繁琐，launch_persistent_context会锁定用户数据目录你自己Chrome就用不了。 最后用了cookie注入，最省事。从Chrome里把知乎的z_c0cookie导出来，这个cookie是知乎登录凭证： [{"name":"z_c0","value":"2|1:0|10:...|4:z_c0|80:...", "domain":".zhihu.com","path":"/","secure":true}] Playwright里一行注入完事： context.add_cookies(json.loads(Path("zhihu_cookies.json").read_text())) 数据全出来了。 还有个小坑顺手说了。知乎内容经常带emoji，Python在Windows终端默认GBK编码，GBK不认emoji，批量跑到带灯泡emoji的标题就直接crash了——UnicodeEncodeError: 'gbk' codec can't encode character '\U0001f4a1'。脚本开头强制UTF-8输出就行： import io, sys if sys.stdout.encoding and sys.stdout.encoding.lower() != 'utf-8': sys.stdout = io.TextIOWrapper(sys.stdout.buffer, encoding='utf-8', errors='replace') 回过头说，这题的关键不在于具体用什么方案——stealth也好cookie也好都不复杂——在于分清楚你卡在哪一层。第一层是网站反爬，检测你浏览器指纹不对，stealth搞定。第二层是Chromium残缺，浏览器自己不行，得绕着走。第二层更坑，因为你不知道它是第二层的问题。
