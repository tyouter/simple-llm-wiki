谷歌提出CLI的7条设计原则，AI和人类都友好的CLI长这样 王鹏LLM​ 华东理工大学 工业催化硕士 11 人赞同了该文章 最近CLI特别火，很多人认为CLI会成为AI调用的工具的最佳方式。但人类也经常用CLI。他们希望交互式提示、彩色输出、终端UI——这些设计标准，对AI无益处。但是砍掉它们，人类体验又变差。其实不用二选一。你的CLI可以同时服务两者。 核心哲学：数据与呈现分离 解决方案藏在一条原则里：把数据从呈现中剥离。 AI需要原始的、结构化的数据（JSON）。人类需要可读的、交互式的格式（TUI）。把CLI内部逻辑看作数据引擎，终端UI只是其中一个渲染客户端。 # 人类看到交互式TUI a2acli watch --task abc123 # AI拿到结构化NDJSON a2acli watch --task abc123 --no-tui 同一个命令，两种渲染器。接下来拆解七个关键设计模式。 1. 结构化可发现性 --help是CLI入口点，人类和AI都从这里开始。 按功能分组：Task Management、Configuration这样的分类，避免文字墙。标记入口点：加上(start here)提示，AI会据此判断先调用哪个命令。 三个必填字段： Short: 5-10词，动词开头 Long: 解释"为什么"和差异化 Example: 3-5个可复制粘贴的例子 例子比描述重要。开发者会复制，AI会解析例子推断flag模式。别假设AI天生知道怎么用你的工具。在仓库根目录放AGENTS.md，定义交互规则、默认工作流。复杂命令加skills/目录，包含专门的提示文件。就像给AI一本用户手册。 2. AI优先互操作性 要让AI能用CLI，输出必须可解析、可预测。 --json或--no-tui:每个产生数据的命令都要支持。输出必须是有效的JSON或NDJSON。AI解析不了，你的CLI在AI世界就不存在。 自动检测受众:支持NO_COLOR和[APP]_NO_TUI环境变量。stdout被管道传输时，跳过交互元素。 非交互式回退机制: 使用用户界面（TUI）的命令（如Bubble Tea）应具有纯文本模式，该模式将标准文本或JSON输出到标准输出（stdout）。TUI是用户界面，而JSON输出则是代理界面。同一个命令，两种呈现方式。 保护上下文窗口: AI有token限制。截断大段文本，屏蔽敏感secret。需要原始payload时，要求--full或--verbose。 预处理数据:自动排序输出，让最关键的项目（严重漏洞、待处理任务）永远在顶部。AI不该写复杂逻辑来找重点。 无状态设计:用引用标识符（--task <ID>）保持CLI无状态。后端维护上下文和对话历史，CLI只是传输机制。 3. 配置与上下文 遵循XDG规范 配置放在~/.config/app/config.yaml。不要在家目录放dotfile。 命名环境 用--env切换配置： envs: local: { service_url: "http://127.0.0.1:9001" } staging: { service_url: "...", token: "..." } prod: { service_url: "...", token: "..." } 编排器设置--env prod，不需要知道URL或token。 4. 错误指导 不要只报错，要指路。 上下文提示： Error: database not initialized Hint: run 'a2acli init' to create the local database AI会解析提示并自我纠正。快速失败：执行重逻辑前先验证配置。别让AI等30秒后发现token缺失。确定性退出码： 0:成功 1:一般错误 2:无效用法 3:认证失败 失败操作返回0，会破坏所有自动化。 5. Flag一致性 语法必须可预测。学会一个命令，直觉要能延伸到整个应用。 标准化shorthands：-o在一个命令里是--out-dir，另一个命令里绝不能是--output。不一致会破坏AI推理。 位置参数 vs flag：位置参数用于必需实体，--flags用于可选修饰符。 安全默认值：破坏性操作必须要求显式flag。 6. 视觉设计 颜色服务于功能，不是审美。 语义颜色token：Accent（地标）、Command（扫描目标）、Pass（成功）、Warn（过渡）、Fail（错误）、Muted（元数据）、ID（标识符）。 规则：保留颜色给状态。绿色=成功，黄色=警告，红色=错误。不要给帮助文本着色。用空白表示层级。支持明暗主题。 7.版本与生命周期 CLI工具不是静态制品。它运行在动态序列中——版本会漂移、操作会被突然中断、安全工作流需要审计轨迹。智能处理这些生命周期事件，能防止自动化系统的静默失败。CLI应该知道自己的版本：支持--version，可选通知用户上次运行后的重大更新。 优雅处理中断：SIGINT（Ctrl+C）时不能损坏数据——当有人或AI杀死运行中的操作时。 追踪操作者：写操作上记录"Actor"（从git config user.name或$USER推导），确保可审计。 结语 数据与呈现分离是CLI设计的终极形态。 前后端分离、API与UI分离，我们见过无数次。但应用到CLI这个"古老"工具上，却是一种顿悟。 CLI的本质是什么？命令行接口，数据传输管道。TUI只是可视化层。剥离这两层，问题迎刃而解。人类需要颜色、交互？渲染器A负责。AI需要JSON、结构化数据？渲染器B负责。数据引擎不变。 2026年的CLI不为AI优化，等于不存在。 Cursor、Claude、Copilot等AI正在接管开发工作流。你的CLI迟早被调用。AI解析不了输出，它就去找别的工具。--json、--no-tui、确定性退出码——小成本大收益的投资。 参考 Build a CLI for AI agents & humans - Google Cloud Tech clig.dev - CLI设计最佳实践 编辑于 2026-03-31 14:02・湖北 AI​ AI技术 AI-Agent ​ 赞同 11​ ​ 添加评论 ​ 64 ​ 喜欢 ​ 分享 ​ 申请转载 ​ ​ 未登录用户 还没有评论，发表第一个评论吧 关于作者 王鹏LLM 算法不能解决所有事，但是能解决一些事 ​ 华东理工大学 工业催化硕士 回答 33 文章 141 关注者 4,059 ​ 关注他 ​ 发私信 大家都在搜 换一换 伊朗局势 569 万 热 全红婵含泪恳求别再骂了 410 万 热 浪姐 7 初舞台直播 372 万 热 王楚钦逆转晋级世界杯四强 353 万 新 清明 343 万 热 突发，京东副总裁被开除 334 万 热 天龙三号发射失利 295 万 热 NASA 阿尔忒弥斯 2 号载人绕月飞行 284 万 热 武汉一男子持刀致4人受伤 276 万 张雪称影视版权给顶级团队 230 万 广告 推荐阅读 Flutter 即学即用系列博客——01 环境搭建 安卓小煜 设计模式｜输入线索：如何让用户清楚该输入什么？ 空谷 设计每周清单：菜鸟发布新品牌标识、Sketch原型预览工具Mirr.io、Google I/O 2017 Design Talks sheng... 发表于Desig... 客户端与Linux图形栈 594飞飘
