人工智能，尤其是大型语言模型（LLM）的飞速发展，为包括地理信息表示和计算在内的各个领域开辟了全新的可能性。这种技术融合正在推动地理信息科学（GIScience）发生范式转变，促使地理信息系统（GIS）朝着自主化的方向发展，并彻底改变了传统的工作流程。LLM 理解、生成和推理人类语言的能力，为地理领域及其与空间信息的集成带来了变革性的潜力。 自主地理信息系统：下一代 GIS 自主地理信息系统（Autonomous GIS）被认为是下一代人工智能驱动的 GIS，它利用 LLM 作为推理核心，通过自动化的空间数据收集、分析和可视化来解决空间问题。自主 GIS 的概念框架包括五个自主目标、五个自主级别、五个核心功能和三个操作尺度。五个自主目标描述了自主 GIS 期望的行为：自生成（创建假设、研究思路、地理处理工作流程、代码和见解）、自执行（准备数据并执行地理处理工作流程以产生结果）、自验证（验证其每一步操作和最终输出的结果）、自组织（有效管理和分配资源，包括时间、计算能力和通信）以及自成长（不断从过去的任务和外部文档中学习以提高未来的性能）。 通往完全自主化的结构化路径由五个自主级别表示，其灵感来源于自动驾驶汽车的级别：Level 0：手动 GIS（无自动化），所有任务都由用户手动执行；Level 1：例行感知 GIS（过程自动化，人为驱动），自动化预定义的重复性任务，工作流程由人定义；Level 2：工作流程感知 GIS（工作流程生成，人为辅助），基于用户准备的数据，为给定的任务自主生成和执行工作流程；Level 3：数据感知 GIS（数据准备自动化，人为监督），在人为监督下自动化数据收集和准备，处理数据不确定性；Level 4：结果感知 GIS（结果解释，极少监督），基于结果评估优化工作流程和数据使用，人为监督极少；Level 5：知识感知 GIS（自成长，无需监督），完全自主的系统，能够无需人为监督地学习和提高性能。 当前的研究主要集中在 Level 2 的智能体上，这些智能体能够基于自然语言输入自主生成和实施空间工作流程。早期的实现包括 LLM-Find（自然语言驱动的数据检索）、LLM-Geo（用于空间分析的工作流程生成）、LLM-Cat（视觉驱动的自主制图）和 GIS Copilot（协助用户使用现有工具进行地形分析的 QGIS 插件）。一个名为 LLM-Geo 的原型系统，使用 GPT-4 API，展示了自主 GIS 在诸如统计危险废物附近的人口、分析人口流动数据以及可视化 COVID-19 死亡率等任务中的潜力。自主 GIS 的概念代表着向非专业人士开放空间分析的重要一步，它通过抽象掉传统 GIS 软件的复杂性，使得地理空间分析更加易于使用。诸如 LLM-Find、LLM-Geo 和 LLM-Cat 等概念验证智能体的开发，突显了自主 GIS 不同方面的可行性，这表明构建此类系统可以采取模块化的方法，首先解决各个组成部分。 以下表格总结了自主 GIS 的五个自主级别： 自主级别 描述 0 手动 GIS (无自动化) 1 例行感知 GIS (过程自动化，人为驱动) 2 工作流程感知 GIS (工作流程生成，人为辅助) 3 数据感知 GIS (数据准备自动化，人为监督) 4 结果感知 GIS (结果解释，极少监督) 5 知识感知 GIS (自成长，无需监督) LLM 增强地理空间数据管理 LLM 可以作为自然语言驱动的数据检索代理（例如 LLM-Find），能够根据用户查询访问包括 OpenStreetMap 和 Esri World Imagery 在内的各种数据源。LLM-Find 框架利用数据源索引和包含每个数据源元数据和检索细节的手册，实现自主数据获取。该框架已作为 QGIS 插件（GeoData Retrieve Agent）和 Python 程序进行了原型设计，在从各种来源检索数据方面取得了很高的成功率。LLM 还可以分析来自报告、社交媒体和历史记录的非结构化文本，以发现空间洞察力，并通过解决地名和本体中的歧义来改进数据集成。通过将 LLM 与矢量存储解决方案和第三方 API 集成，GeoAI 应用程序可以高效地访问、提取和合成来自庞大数据集的地理空间相关内容。LLM 理解自然语言查询的能力显著降低了访问地理空间数据的门槛，使得用户更容易找到他们需要的信息，而无需了解特定的数据库结构或查询语言。诸如 LLM-Find 等框架的即插即用架构展示了一种社区驱动的方法来扩展自主 GIS 代理可访问的数据源。用户可以通过创建手册来添加新的数据源，这种方式促进了协作，并允许系统适应不断增长的地理空间数据环境。 智能空间分析和工作流程自动化 LLM 可用作工作流程生成代理（例如 LLM-Geo），根据自然语言请求自主执行地理处理工作流程，这已在学校步行可达性评估中得到证明。LLM-Geo 原型采用分而治之的策略，将复杂的任务分解为更小、更易于管理的操作，并使用 GPT-4 API 生成 Python 代码。LLM 可以解释与地理空间数据相关的自然语言查询，执行语义解析以将这些查询转换为 GIS 工具可以理解的结构化、机器可读的格式。它们可以转换空间术语并处理复杂的空间关系，从而使地理空间分析更加直观。诸如 ChatGeoAI 等框架利用微调的 LLM（例如 Llama 2）和地理空间本体来理解自然语言查询，生成可执行的 PyQGIS 代码，并为 GIS 技能有限的用户执行地理空间分析。LLM 自动化空间分析工作流程的能力，有可能显著提高效率并缩短复杂地理空间调查所需的时间。在传统的 GIS 中，手动设计和执行空间分析工作流程可能非常耗时且需要专门的知识。LLM 可以自动化此过程，使用户能够更快地获得结果。使用特定领域的本体和微调技术可以增强 LLM 准确解释地理空间查询并生成相关分析代码的能力。通用 LLM 可能缺乏 GIS 概念和术语的特定知识。通过对地理空间数据进行微调并整合本体，可以提供必要的领域专业知识。 LLM 驱动的制图革命 LLM-Cat 是一种视觉驱动的代理，它使用 GPT-4o 的功能根据自然语言请求生成地图，并迭代地改进地图设计。MapGPT 等框架利用自然语言处理来理解用户需求，并调用各种制图工具来生成地图元素，从而简化了制图过程。制图通过为用户提供可视化复杂数据、提取有意义的见解和有效传达其发现的方式，成为 GeoAI 和 LLM 的关键促成因素。LLM 可以分析文本数据以提取地理空间相关信息，然后可以通过地图进行地理编码和可视化，从而增强灾难响应等应用中的态势感知能力。LLM 与制图工具的集成可以普及地图创建，允许用户根据自然语言描述生成可视化效果，而无需广泛的制图专业知识。传统的地图制作涉及理解制图原则和使用复杂的 GIS 软件。LLM 可以通过解释用户请求并自动化设计选择来简化此过程。LLM 处理文本信息并将其转换为地图要素的能力突显了创建动态和上下文感知可视化的潜力。通过理解文本的语义内容，LLM 可以创建不仅视觉上吸引人而且有效地传达底层信息的地图。 自然语言：新的 GIS 界面 LLM 越来越多地用于解释与地理空间数据相关的自然语言查询，从而使 GIS 更易于经验不足的用户使用。这些模型可以将自然语言转换为 GIS 工具或数据库可以理解和操作的结构化、机器可读的格式（例如 SQL、OverpassQL、CQL）。示例包括使用 LLM 为关系数据库生成 SQL 查询以及为 OpenStreetMap 数据生成 OverpassQL 查询。ArcGIS Hub AI Assistant 利用 Esri 管理的 LLM，使用通过 ArcGIS Hub 站点共享的开放数据，将用户请求转换为地理空间查询和摘要。由 LLM 驱动的自然语言界面可以显著提高 GIS 的可用性，使其能够被更广泛的用户群体（而不仅仅是训练有素的 GIS 专业人员）所使用。通过允许用户使用熟悉的语言与 GIS 进行交互，LLM 消除了学习复杂软件界面和特定命令的需要。LLM 与 ArcGIS Hub 等平台的集成展示了创建智能界面的潜力，这些界面可以理解用户需求并直接提供相关的地理空间信息。这种集成可以通过使公众和其他利益相关者更容易访问和理解开放数据来增强其价值。 从非结构化数据中提取地理空间知识 LLM 可以分析大量文本数据，以识别和提取地理空间信息，例如地名、空间关系和地理特征。通过分析社交媒体、新闻文章和历史文献，此功能对于地理空间趋势分析、灾害监测和基于舆情的城市研究等任务非常有价值。POI GPT 将命名实体识别与 LLM 相结合，从文本中提取精确的 POI 位置，从而降低了传统方法的成本和时间。微调的 LLM 可以在从非结构化社交媒体文本中识别和提取结构化灾难实体及其属性方面实现高精度。LLM 可以将大量的非结构化文本数据转化为有价值的地理空间情报，提供仅通过传统 GIS 方法可能难以或不可能获得的见解。处理和理解文本的能力使 LLM 能够挖掘文档、文章和社交媒体帖子中包含的大量信息，从而丰富我们对空间现象的理解。诸如 POI GPT 和灾难信息提取系统等专用 LLM 应用程序的开发，展示了针对特定地理空间用例从文本中进行有针对性的知识提取的潜力。通过专注于特定类型的信息，这些专用应用程序可以在提取相关的地理空间知识方面实现更高的准确性和效率。 以下表格总结了 GAI 模型在地理空间科学中的不同角色： LLM 与现有 GIS 平台的集成 ESRI 正在将生成式 AI 和 LLM 集成到 ArcGIS 中，例如 EuclidHL 可以简化对城市规划和分区信息的访问。ArcGIS Pro 允许通过自定义 NLP 功能和深度学习包与第三方 LLM（开源或云托管）集成，用于地址标准化和文本分类等任务。ArcGIS Hub 具有由 Esri 管理的 LLM 驱动的 AI 助手，用于提供来自开放数据的答案和摘要。QGIS 插件（如 LLM-Find (GeoData Retrieve Agent)、Promptly 和 IntelliGeo）正在开发中，以实现使用 LLM 进行自然语言交互、代码生成和工作流程自动化。ChatGeoAI 是另一个 QGIS 集成，它使用微调的 Llama 2 模型根据自然语言查询生成用于地理空间分析的 PyQGIS 代码。针对 ArcGIS 和 QGIS 的 LLM 集成的积极开发表明，将自然语言功能纳入标准 GIS 工作流程已成为一种强劲的趋势。这些集成旨在通过允许用户使用自然语言命令和查询与软件进行交互，从而使 GIS 更加用户友好和高效。各种集成方法（从内置助手到自定义插件）表明，LLM 如何集成到 GIS 平台是一个灵活且不断发展的领域。不同的用户需求和技术能力正在通过各种集成方法得到满足，这反映了对在 GIS 中利用 LLM 的最佳方式的持续探索。 评估 LLM 在地理空间任务中的性能和可信度 LLM 在地理空间任务中的性能仍有待探索，因此需要开发专门的评估数据集和基准。一个涵盖十二种任务类型（例如空间理解、路径规划）的多任务空间评估数据集已被创建，用于评估 LLM 的性能。地理空间任务的评估指标包括准确度评分（完全正确、部分正确、不正确）以及用于代码生成（例如 CodeBLEU）和 SQL 查询性能（例如完全匹配、有效效率得分）的专门指标。GeoBenchX 和 GEOBench-VLM 等基准分别专门用于评估 LLM 在多步骤地理空间任务和视觉语言模型在地理空间任务中的性能。评估 LLM 在地理空间分析中的挑战包括需要特定领域的知识、处理复杂的空间和拓扑关系以及减轻幻觉。专用地理空间基准和评估指标的创建标志着人们越来越认识到在地理空间领域使用 LLM 的独特挑战和要求。通用 LLM 基准可能无法充分评估 GIS 应用所需的空间推理和数据处理能力。不同 LLM 模型在地理空间任务中的不同表现突显了为特定应用选择合适的模型并使用有效的提示策略的重要性。并非所有 LLM 都同样擅长空间推理或处理地理空间数据格式。基准测试有助于识别不同模型的优势和劣势。 地理基础模型的出现 地理基础模型（GeoFMs）是专门针对大量地理空间数据（例如卫星图像、地形图）进行训练的大规模深度学习模型。这些模型学习地理空间数据的广义表示，使它们能够在最少的微调下在多个下游任务中表现良好。示例包括 SatMAE（用于卫星图像识别）和 Prithvi（在 Landsat 和 Sentinel-2 图像上训练，用于土地覆盖制图和其他任务）。谷歌研究院正在开发新的地理空间 AI 基础模型，这些模型在卫星和航空图像上进行训练，以支持关于地理空间特征的自然语言查询。Niantic Labs 正在开发大型地理空间模型，以使计算机能够基于数十亿个地理参考图像来感知、理解和导航物理世界。GeoFM 的开发代表着创建对地理空间数据的复杂性有更深入理解的 AI 模型的重要一步，这可能在各种应用中带来突破。专门针对地理空间数据训练模型可以比通用 LLM 更有效地学习空间模式和关系。谷歌和 Niantic 等主要科技公司参与开发 GeoFM，凸显了该领域日益增长的重要性和潜力。这些行业领导者的投资和研究可能会加速 GeoFM 在各个领域的开发和采用。 跨越不同 GIS 领域的应用 LLM 在环境监测、灾害管理、城市规划和考古研究等领域的具体用例展示了 LLM 在各种 GIS 领域的通用性和潜在影响，解决了不同的挑战和用户需求。 环境监测： LLM 可以分析卫星数据、传感器读数和文本报告，以监测森林砍伐、空气质量和其他环境趋势，从而帮助进行资源管理和保护。 灾害管理： LLM 可以处理来自社交媒体、新闻媒体和传感器的实时数据，以提供态势感知、支持决策制定并协助灾害响应和恢复工作。 城市规划： 集成 GIS 的 LLM 可以帮助解释土地使用法规、识别合适的开发地点并回答市民关于规划和分区的疑问。 考古研究： LLM 可以通过在考古数据集上进行训练来增强其解释和复制常见地理空间分析的能力，从而简化研究并自动化复杂的空间数据任务。 交通运输： LLM 可用于交通运输决策、分析交通模式以及提供来自交通运输数据集的见解。 这些广泛的应用领域表明，LLM 在各种 GIS 领域都具有多功能性和潜在的影响，能够解决不同的挑战并满足用户的需求。这些具体应用的示例突显了 LLM 与 GIS 集成的实际价值，从理论概念走向现实世界的问题解决。 挑战、局限性和未来方向 当前的挑战包括 LLM 缺乏足够的 GIS 特定知识、难以处理复杂的空间推理和拓扑关系以及潜在的不准确性和偏差。当将 LLM 用于大规模 GIS 处理时，计算成本和可扩展性也是需要考虑的问题。未来的研究方向包括增强地理空间数据查询和推荐系统、构建基于 LLM 的强大评估框架、开发用于 GeoAI 的多模态基础模型以及向 GeoAGI（地理通用人工智能）迈进。提高 LLM 的空间编程技能、为自主 GIS 开发记忆系统以及实现自主地理建模也是未来工作的关键领域。虽然 LLM 为 GIS 提供了巨大的潜力，但仍存在一些技术和实践挑战需要通过持续的研究和开发来解决。克服空间推理、数据处理和可信度方面的局限性对于 LLM 在 GIS 中的广泛采用至关重要。研究人员概述的未来方向预示着 LLM 和 GIS 将走向更加复杂和集成的未来，并有可能实现真正智能和自主的地理空间系统。GeoAGI 的愿景以及对多模态模型的关注表明，一个长期的目标是创建能够深入理解和与空间世界交互的 AI 系统。 LLM 与 GIS 的集成正在彻底改变地理空间数据的访问、分析、可视化和理解方式。由 LLM 驱动的自主 GIS
