OpenClaw 收录于 · 折腾党 448 人赞同了该回答 网安狗来啦，我来给大家整理下OpenClaw新手注意事项： 1、有条件建议单独拿台电脑装OpenClaw，没条件也不要装在办公主机上，而是装在虚拟机里； 2、及时观察Token消耗，避免一下子扣太多费用； 3、不要自己创造需求，比如：看到别人玩OpenClaw自己啥也不懂就下单个Macmini; 4、不要把自己的ApiKey、邮箱凭证、各种密钥文件放在workspace文件夹； 5、不要一开始就充收费的Api，可以先注册NVIDIA Build获取免费不限量的api； 6、只安装自己能看懂的skill，看不懂的不要一股脑全选安装； 7、不要为了让OpenClaw能识图，就花钱买有读图功能的大模型，会产生不必要的消费； 8、建议先玩一下Napcat+AstrBot的QQ机器人，这个能满足90%人需求且部署简单，如果能满足你的需求那就不用折腾OpenClaw了； 9、不要考虑让OpenClaw使用你的微信账号，很多人执念这个，风险极大； 10、不要将自己已有账号权限给OpenClaw，而是需要为OpenClaw注册新的账号，也不要将自己的OpenClaw的Bot分享给别人，因为这个Bot有你电脑的完整权限，你要确保这个Bot只能响应你自己的消息； 11、不要将OpenClaw安装在已经运行容器或其他服务的环境，因为操作不当会删掉容器或影响服务环境； 12、你要了解OpenClaw的心跳是干啥的，初步理解就是【每隔一会把AI叫起来看看有没有需要做的事情的功能】，如果你想不到这个功能能帮你做什么事，那你就不需要OpenClaw； 13、买大模型API的话，不要去闲鱼买那种中转API的，售后不太行； 14、如果你喜欢折腾，而恰好电脑有32G以上内存或显存，可以先不用买API，研究下自己电脑跑GLM-4.7-Flash-Q4； 人赞同了该回答 许多人应该都知道， open claw 最近火遍全球了。可以说，2026 年的趋势已经很明显了，就是 AGENT 元年，很多主流模型和应用都在做 AGENT概念的落地。openclaw 是其中一个有明星效应的标杆级作品。大神们拿着 openclaw 玩出了各种花。对于具体应用场景，也没啥好多说的了，因为每个人关注的领域不同，别人眼里的香饽饽，可能你觉得就是路边野菜。所以我只为新手小白们稍微说几句我在安装过程中的折腾经验，帮助小白们在初期少走弯路。我在安装中遇到的坑也是踩了又踩，还真不少。我总结了一下，给几点经验和建议吧， 1，建议用虚拟机装这只充满了挑战和风险的大龙虾！将测试环境和真实使用环境隔离开，是最要紧的事。我自己用丐版的MAC mini 4 部署，15.7.3 系统。当然有些人喜欢云端部署，也是可以的。本地部署的最大好处就是调试及时，方便。虚拟机部署完了可以克隆分发，免去了反复装的麻烦。一定要考虑部署云端的话，要注意你用的是原版龙虾(openclaw)还是平行宇宙版的各种龙虾（nonoclaw,zeroclaw......）。因为各种龙虾对容器的环境要求不同。仅以我个人经验来看，原版龙虾的话，实测内存大约需要2-3G内存，如果配置了更多的工具，内存会增加到6G，云端是否能满足？要提前了解下。云端是否支持备份或者快照机制也需要事先了解。云端的权限也是个问题，当系统装完后，会经常需要加装一些skills技能，有些技能会要求的系统权限比较大，在云端的沙盒环境下去跑就会受限，这是系统底层的限制，无法突破。所以，有条件的话，还是建议本地安装，操作系统首选MacOS，其次Linux，然后Windows+WSL2环境。最后强调一点，现阶段如果要完整地体验龙虾的全部能力，只能是MACOS，其他系统多多少少都有各种限制。 我介绍下 MAC 系统下的虚拟机，我用过 2 个，VirtualBuddy（免费） 和Parallels desktop 26 pro（收费）。前者功能精简，直接一键安装苹果官网 MACOS系统。VB 的缺点是虚拟机容量不能扩容，没有快照功能。PD26 则弥补了前者的不足，支持容量自动扩展，快照也好用，几秒钟拍一个快照，几秒钟还原，这份体验绝对是 MACOS虚拟机平台里最顶尖的。快照功能，就相当于是打游戏用的 S/L 大法（Save & Load），不小心改错了 JSON配置文件，不小心删了啥重要文件，都没事，一键回到安全的起点，绝对省心省力。PD20 也可以用，价格便宜些，只是快照功能没 PD26 优化得这么好。还有一款 VMware Fusion也是支持 MAC 的著名虚拟机软件，对于个人用户免费，商业版收费。但就我所知，免费版提供基础的快照功能，但阉割了一些高级的快照功能，所以我对免费版也不想再多试了。而且 MAC 虚拟机镜像在 parallels 和 VMware 之间格式互转很难，所以要折腾的话，一开始就要想好入哪个门，否则搞了一半想换虚拟机平台了，就难了。其他 MAC 适用的虚拟机平台还有不少，重点关注虚拟机是否支持容量扩展和快照功能，其余功能都是可有可无。 2，小白如何快速上手大龙虾呢？有个技巧。如果你知道怎么用谷歌的 NOTEBOOK LM 的话，就把大龙虾的 GitHub 网址，和官网文档的网络链接作为source都扔进去（谷歌pro付费用户最多300source上限），做个知识库。有问题就和 GEMINI直接对话（要同时挂上知识库），GEMINI 会快速告诉你答案。不会改的配置文件，把 JSON 文件内容贴进去直接提问，就能获得改好的配置文件，再贴回大龙虾系统里就行了，高效快捷。需要注意的是，为笔记添加 source 有一些技巧，一个个手动添加 source url比较费力 (官网有 100 多个文档 source）。具体如何做，可以去参考小红书的一些 notebooklm+openclaw 的帖子。上述这些话是说给小白们听的，高手就免了，直接上手改就得了。 3，虚拟机里 MACOS的 APPLE ID，千万别登录，否则同步一堆个人的信息和资料过来，占了空间不说，私密文件保护也很难处理，除非你非常懂系统权限设置。另外，虚拟机的网络连接问题，PD26 的话，装完默认是与 MAC 机共享局域网连接的，如果你用的是特殊的科学手段，可能就需要改成桥接模式，把虚拟机当成一个挂在局域网上的单机去处理，否则有些网站连不上。 4，小心使用诸如openclaw onboard,doctor --fix等命令，经常会覆盖掉配置文件内容，把原来运行正常的配置文件改得面目全非。如果没有快照，就等着哭吧，修复到再次能用是个痛苦的过程。这也就是为什么极其需要虚拟机软件有快照功能来保驾护航的原因。有快照功能，新手 1-2天搞定基础安装，没有快照功能，折腾一周或更久才能慢慢熟悉起来。龙虾如果要升级，建议分三步走，终端直接敲命令进去，openclaw update --channel stable --> openclaw doctor -->openclaw gateway restart , 迄今为止每次升级都成功，还没翻车过。 5，openclaw 内置的浏览器控制功能可用，但不够强大，建议playwright MCP或者CLI 配套装一个。但请注意，绝大多数 MCP 必须通过 MCPORTER+修改配置文件的方式去调用，这是符合官方规范的安装 MCP 方式。通过MCPORTER 调用的缺点是有一些技术局限，一些MCP 服务选项不是全部支持。期待大龙虾能早日内置全功能的 MCP支持就更好了。所以你在装任何 MCP 时，请参考如下提示词，“我想安装一个 XXX MCP 服务，请按照 openclaw 官方文档中的规范来安装，通过MCPORTER 调用。需要修改配置文件时，不要修改或删除配置文件中的无关内容。”这样的话，安装 MCP 服务的成功率是接近百分百的。否则的话，安装方式有时就会五花八门，系统自启动时后台启动调用的，或者擅自加全局变量的，搞得乱七八糟。 6，消耗 TOKEN 比较大的问题，有多种解决办法。第一个办法，当然是钱包支持，直接买合适的 API 套餐，比如我常用的主模型是 Minimax Coding Plan，49 元一个月，5 小时重置流量，token 在 5 小时内使劲用也没用满过。子模型 AGENT 可以考虑用 Codex 或者Opus，我用的是 19 元 900 美刀的 CODEX 反代API的包月套餐，模型可以支持到最新的 gpt-5.3-Codex。Minimax 应付日常会话，普通任务基本能胜任，但重活干不了因为智商摆在那儿，一分钱一分货。大龙虾可以设置模型自动回滚 FALLBACK 到 Codex 或 Opus ，也就是说 Minimax 解决不了的难题，或者模型挂了，可以让 Codex 来接手。也可以不设子模型 AGENT，直接装 Codex CLI，直接用 PTY 方式去后台调用，用来解决龙虾系统运行故障等复杂问题。编程建议还是单独用Claud Code CLI或者Codex app，这是我近期比较喜欢的软件。第二个省 token 的办法，是启用大龙虾内置的 QMD，可以大幅度减少 token 消耗，具体怎么启用可以直接与模型对话，并参考官方技术文档。 需要提醒的是，minimax M2.5由于模型先天的参数量就不大，能力有限，所以为了让它尽量少犯错，建议开启/think high模式，来跑日常的杂活。现在大龙虾最佳的模型搭档，比较公认的主流看法，是最好搭配opus 4.6或者codex 5.3，这才是真正能发挥龙虾实力的模型。 7，channel 的话，国外推荐用Discord/TG，国内用户用钉钉，目前支持都已经比较成熟。Discord 支持多用户多代理，可以把大龙虾分享给不同的用户来用，每个BOT的频道消息做隔离。如果纯自己用，那么TG就够了，个别功能支持比Discord还好一些。对于国内用户来说，钉钉内置了语音转文字后发送，交流姿势更舒服。TG 的配置比较方便，3-5分钟搞定，钉钉稍微繁琐一些，第一次设置没个 15分钟搞不定，要先官网注册开发者身份。imessage 也可以实现，但步骤也挺多，也不是太建议。未来官方也会推出apple watch应用，交互方式比较新颖。有编程能力的，也可以自己未来去适配智能眼镜，可玩性也很大。 8，网关丢失连不上，在初期安装和调试过程中是经常的事。我建议是让 AI自己做个关键的系统配置文件备份和还原脚本，随时备用。但如果真遇上网关挂了，解决办法也是有的。1，丢失了连接 token，这个在 openclaw.json 文件里可以去自己找回来填上就可以了。2，终端里执行openclaw gateway stop + openclaw gateway start,或者openclaw gateway restart，openclaw gateway install都可以试试。在备份好openclaw.json 的前提下，用openclaw doctor --fix，有时也可以修好并重启网关。3，实在搞不懂咋整，最后就只能靠快照还原了。 9，安全问题，不用说，目前的现状就是大龙虾全身都是漏洞，几百个漏洞等着排队修补呢，所以要装虚拟机里用，不要在公网暴露地址是个原则。装完后，尽早给它定规矩，什么危险动作不能执行，什么问题要请示，等等诸如此类的安全规则，先写进记忆里。装 SKILLS 技能时也要当心，这个是双刃剑，许多技能就是来投毒的，已经有不少案例了。 最后，我必须说一句，当你用心伺候好这只大龙虾，让它能活蹦乱跳开始 24 小时不眠不休地给你干脏活累活的时候，那种惊喜和满足感是无与伦比的！你见证了 AI AGENT在离你工作生活最近的地方开始萌芽和生长，这是一个新的世界，通向未来的大门。 编辑于 2026-03-04 19:19 ​ 赞同 116​ ​ 24 条评论 ​
