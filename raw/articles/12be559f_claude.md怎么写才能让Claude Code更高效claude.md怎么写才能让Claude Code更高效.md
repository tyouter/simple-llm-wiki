CLAUDE.md 用户手动编写的“永久说明书”，是 Claude 跨会话记忆的核心载体。每次新开对话，Claude 会自动读取该文件，无需用户重复告知项目相关规则，主要包含： 项目架构：目录结构、核心模块位置； 编码规范：缩进、命名、注释要求； 工作流程：打包、测试、部署命令； 个人偏好：常用工具、依赖管理方式等。 3. 自动记忆（Auto Memory） Claude 自动学习并记录的“操作笔记”，无需用户手动干预。它会从对话、代码修改、用户反馈中提取有价值的信息，下次会话自动复用，主要记录： 用户偏好：如常用 npm 还是 pnpm； 问题解决方案：某个报错的具体处理方法、项目坑点； 操作习惯：常用调试工具、命令组合等。 4. Subagent（子代理） Claude 的“分身/专属小助手”，专门负责某一类细分任务，如代码搜索、代码重构、文档生成等。每个子代理可拥有独立记忆，互不干扰，能针对性提升特定任务的处理效率。 5. Rules（规则） 当 CLAUDE.md 内容过多、维护不便时，可将其拆分为多个细分规则文件，统一放在 .claude/rules/ 目录下，常见分类包括： code-style.md：编码风格规范； testing.md：测试流程规范； security.md：安全开发要求。 可通过配置限定规则的生效范围，避免规则乱套用。 6. Frontmatter 写在规则文件顶部的配置项，用 --- 包裹，用于限定规则的生效文件范围。示例如下： --- paths: - "src/api/**/*.ts" --- 上述配置表示，该规则仅对 src/api 目录下所有的 TypeScript 文件生效。 7. Hook 类似“事件触发机制”，可在特定场景下自动执行预设逻辑，用于实现自动化配置，常见触发场景包括：指令加载完成后、文件修改完成后、权限请求前等。 8. /memory 命令 Claude Code 会话中的专属查看命令，输入 /memory 可实现以下功能： 查看当前会话加载的 CLAUDE.md 和规则文件； 浏览自动记忆中记录的所有内容； 一键开启/关闭自动记忆功能； 直接打开记忆文件夹，手动编辑或删除不需要的记忆内容。 二、核心对比：CLAUDE.md vs 自动记忆 很多用户会混淆 CLAUDE.md 与自动记忆的功能，两者互补共生，核心区别如下表所示： 对比维度 CLAUDE.md 自动记忆（Auto Memory） 创作者 用户手动编写 Claude 自动学习 核心内容 项目规则、编码规范、架构说明 用户习惯、项目坑点、解决方案 生效范围 全局/项目级，可手动指定 项目级，每个项目单独记忆 加载方式 每次会话全量自动加载 每次会话加载前200行，按需读取 用途 主动给 Claude 定规矩、立标准 被动学习，让 Claude 贴合用户习惯 维护成本 需手动编写、更新 无需手动维护，Claude 自动管理 三、持久记忆体系详解 Claude Code 的持久记忆体系主要由 CLAUDE.md、Rules 规则目录、自动记忆三部分组成，三者协同工作，实现跨会话记忆功能。 1. CLAUDE.md 配置详解 （1）存放位置（按优先级排序，越具体越优先） 根据使用范围，CLAUDE.md 可放在3个位置，适配不同场景需求（以 macOS 为例）： 用户级（全局生效，所有项目通用）：~/.claude/CLAUDE.md，最常用，适合存放个人编码偏好、通用工作流； 项目级（仅当前项目生效，可团队共享）：./CLAUDE.md 或 ./.claude/CLAUDE.md，可提交到 Git，统一团队开发规范； 组织级（公司/团队全局生效）：/Library/Application Support/ClaudeCode/CLAUDE.md，由管理员部署，统一公司编码标准。 （2）编写技巧（避坑关键） 控制长度：单份 CLAUDE.md 建议控制在200行以内，过长会占用上下文，降低 Claude 遵守度； 结构清晰：用标题、列表分组，避免大段密集文字，便于 Claude 识别； 指令具体：明确写出“缩进用2空格”“提交前运行 npm test”，避免模糊表述（如“代码格式规范”“测试代码”）； 导入复用：用 @路径 导入 README、package.json 等文件，无需重复编写，示例：@README、@docs/git-workflow.md。 2. Rules 规则目录配置 当项目规模较大、CLAUDE.md 内容过多时，可将规则拆分为多个细分文件，放在 .claude/rules/ 目录下，便于维护和管理。 核心优势：可通过 Frontmatter 配置，让不同规则仅作用于特定类型的文件，提升规则的精准度。 3. 自动记忆配置 （1）存储位置 自动记忆文件默认存放在 ~/.claude/projects/<project>/memory/，每个项目单独存储，互不干扰，主要包含两类文件： MEMORY.md：记忆索引，每次会话仅加载前200行，避免占用过多上下文； 主题文件：如 debugging.md（调试笔记）、api-conventions.md（API 规范），详细记录特定领域的记忆内容。 （2）开关与管理 自动记忆默认开启，可通过 /memory 命令查看、编辑、关闭，也可在 settings.json 中手动配置。 四、权限模式详解 权限模式用于控制 Claude 执行操作前是否需要用户确认，不同任务对应不同的权限需求，合理选择模式可兼顾效率与安全。Claude Code 提供6种权限模式，核心常用的有5种，按“安全→高效”排序如下： 1. 核心权限模式详解 模式名称 功能解读 适用场景 操作方式 default（默认模式） 所有操作（读文件、改文件、跑命令）均需用户手动确认 新手入门、敏感工作（如操作生产环境代码） 默认启动，无需额外操作 acceptEdits（自动接受编辑） 可直接读、改文件（无需确认），跑命令、联网仍需用户确认 日常写代码、快速迭代、修改本地代码 CLI 按 Shift+Tab 切换，或启动时加 --permission-mode acceptEdits plan（规划模式） 仅读文件、分析代码、出方案，不修改任何内容 探索陌生代码库、大型重构前做方案、避免误改代码 启动时加 --permission-mode plan，或会话前加 /plan auto（自动模式） 几乎所有操作自动执行，后台有安全分类器把关，阻止危险操作 长时间任务（如批量重构）、不想被弹窗打扰 需 Team 计划+Claude 4.6+，CLI 加 --enable-auto-mode 后切换 bypassPermissions（跳过所有权限） 所有操作无任何检查，Claude 拥有完全控制权（极度危险） 仅隔离环境（虚拟机、沙盒），测试危险操作 启动时加 --permission-mode bypassPermissions 2. 关键补充：Auto 模式（高效常用） Auto 模式是日常开发中最常用的高效模式，使用时需注意以下几点： 安全保障：后台有分类器（安全AI），默认阻止危险操作（如删库、curl | bash、强制推送 git、部署生产环境）； 默认允许：本地文件编辑、安装锁定文件中声明的依赖、只读网络请求； 回退机制：连续3次被阻止后，Auto 模式自动暂停，退回手动确认，避免误操作； 成本与延迟：分类器会消耗少量令牌，每个操作有轻微延迟，可忽略不计。 3. 模式切换技巧（全场景适配） CLI/JetBrains：按 Shift+Tab 循环切换模式，当前模式显示在状态栏； VS Code：点击提示框底部的模式指示器，或在扩展设置中设置默认模式； Desktop/Web：使用模式选择器（发送按钮旁边），会话中可随时切换； 设为默认：在 settings.json 中配置 "permissions": {"defaultMode": "acceptEdits"}，每次启动自动生效。 五、保姆级实操配置步骤（macOS 环境） 本配置适配 macOS 系统，配置根目录为 /Users/jiangyunchao/.claude，适用于 Python/AI 大模型/前端/工程化开发，将完整搭建持久记忆体系与权限模式，可直接复制操作。 准备工作 打开终端，进入配置根目录： cd ~/.claude 步骤1：创建用户级 CLAUDE.md，配置全局规则 创建 CLAUDE.md 文件： touch CLAUDE.md 编辑文件（可用 VS Code、记事本打开），粘贴以下内容（可按需修改）： # 个人 Claude Code 全局配置（用户级） ## 一、编码规范 ### 1. 通用规范 -
