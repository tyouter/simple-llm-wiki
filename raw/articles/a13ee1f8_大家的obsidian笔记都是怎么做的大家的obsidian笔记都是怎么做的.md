Obsidian 忽略规则 .obsidian/plugins/ .obsidian/themes/ .obsidian/workspace .obsidian/graph.json .obsidian/hotkeys.json .obsidian/app.json .obsidian/core-plugins.json .obsidian/community-plugins.json # 缓存和临时文件 *.tmp .DS_Store Thumbs.db # 如果你不想同步附件大文件（可选） # attachments/*.png # attachments/*.jpg 说明： 保留 .obsidian/appearance.json、snippets/ 等如果你希望同步主题或 CSS 片段。 不要忽略整个 .obsidian/ 目录，否则会丢失重要配置（如核心插件开关、笔记关系图设置等）。 插件和主题通常体积大且可重新安装，建议忽略。 步骤 3：首次提交 git add . git commit -m "Initial commit: Obsidian vault setup" 步骤 4：连接远程仓库（可选但推荐） git remote add origin git@github.com:yourname/your-vault.git git push -u origin main 隐私提醒： 如果笔记含敏感信息（如密码、个人日记），不要使用公开仓库！ 推荐： GitHub/Gitee 的 私有仓库 自建 Git 服务器（如 Gitea） 或使用加密方案 安全与隐私增强方案 方案 A：使用私有仓库 GitHub / GitLab / Gitee 均提供免费私有仓库。 确保仓库设为 Private。 方案 B：本地 Git + 加密同步（高安全） 用 git-crypt 或 git-secret 加密敏感文件后再推送到远程。 或将整个 vault 存放在 加密网盘（如 Cryptomator + Dropbox）+ 本地 Git。 **### 方案 C：仅同步非敏感笔记 将笔记分为 public/ 和 private/ 目录； 在 .gitignore 中排除 private/； 敏感内容只保留在本地。 注意事项 Obsidian 不会自动触发 Git 提交 → 需手动操作，或用脚本/工具自动化（如 cron 定时任务、Obsidian Git 插件）。 Obsidian Git 插件（可选） 功能：在 Obsidian 内一键 pull/push/commit。 安装：社区插件搜索 “Obsidian Git”。 ⚠️ 注意：它仍依赖本地 Git 环境，且自动提交需谨慎（可能提交未完成内容）。 不要频繁提交二进制附件 Git 不擅长处理大文件变更，会导致仓库膨胀。 建议：图片等附件单独管理，或使用 Git LFS（但增加复杂度）。 可能遇到的问题 换行符转换 提示信息： “LF will be replaced by CRLF the next time Git touches it” 是 Git 在 Windows 系统上处理换行符（line endings） 时发出的警告。它本身不是错误，而是一个通知，告诉你 Git 即将对文件的换行符进行自动转换。 但对大多数项目（尤其是开源项目），保留自动转换是更安全的选择。 特别注意：某些文件不能转换！ Shell 脚本（.sh）、Dockerfile、Makefile 等在 Linux 上运行的文件必须用 LF。 如果 Git 把它们转成 CRLF，会导致脚本在 Linux 上报错（如 ^M: command not found）。 ✅ 解决方案：在项目根目录添加 .gitattributes 文件，强制指定换行符： # 强制这些文件始终使用 LF *.sh text eol=lf *.py text eol=lf Dockerfile text eol=lf Makefile text eol=lf *.md text eol=lf # 二进制文件不转换 *.png binary *.jpg binary 这样即使 core.autocrlf=true，这些文件也不会被转成 CRLF。 设置身份信息 全局身份信息 在终端中运行以下两条命令（替换为你自己的名字和邮箱）： 注意： 邮箱不需要真实存在，但建议用常用邮箱（如 GitHub 邮箱可关联提交记录）。 如果你用 GitHub，建议邮箱与 GitHub 账户一致，这样提交会自动关联到你的账号。 --global 表示全局设置，以后所有仓库都用这个身份。如果你只想对当前 Obsidian 仓库设置，去掉 --global 即可。 ✅ 正确示例： git config --global user.name "ZhouNB" git config --global user.email "zhounb@example.com" 仅对当前仓库设置（可选） 如果你不想影响其他项目，可以不加 --global，在你的 Obsidian 笔记库目录下运行： cd /path/to/your-obsidian-vault git config user.name "ZhouNB" git config user.email "zhounb@example.com" 这样身份信息只对这个仓库生效。 中文文件名的处理 Git 日志、状态等命令的输出显示：会显示为转义形式（如 \344\270\255\346\226\207.md） 这是因为core.quotepath = true（Git 默认值）在 Windows 下 _不会_ 改变你磁盘上实际的文件名，它只影响 Git 命令行输出中文件名的显示方式。 虽然 core.quotepath=true 不影响实际功能，但对用户不友好（看不懂文件名）。建议： git config --global core.quotepath false 这样： git status、git log --name-only 等命令会直接显示中文文件名； 前提是你的终端支持 UTF-8（现代 Windows Terminal、PowerShell 7+、VS Code 终端都支持）。 推荐日常命令 查看当前配置 你可以随时检查 Git 是否记住了你的信息： git config --global user.name git config --global user.email 或者查看所有配置： git config --list 日常提交 git add . git commit -m "测试中文文件名" 查看日志 场景 命令 快速看最近提交 git log --oneline -10 查看完整变更 git show 理解分支结构 git log --graph --oneline --all 查某文件历史 git log --follow filename 提交到远程仓库 git remote add origin git@github.com:yourname/your-vault.git git push -u origin main 发布于 2026-01-10 04:26 ​ 赞同 11​ ​ 7 条评论 ​ 48 ​ 喜欢 ​ 分享 ​ 收起​ 知乎用户 9 人赞同了该回答 我只能分享我自己的办法，可以作为你的参考。 不过由衷的建议，不要讲自己的沉浸在插件和主题的海洋，记录软件重点还是要记录内容和归纳自己的想法。 #
