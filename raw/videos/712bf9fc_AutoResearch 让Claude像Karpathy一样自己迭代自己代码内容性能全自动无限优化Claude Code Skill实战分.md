## 简介


最近AI圈最火的概念之一，就是Andrej Karpathy开源的那个“autoresearch”——简单来说，就是让AI自己设定目标、自己做实验、自己跑loop、自己保留好的结果、丢掉差的，睡觉的时候它还在疯狂进步，醒来就能看到指标暴涨。
但Karpathy的版本主要针对机器学习训练循环。那有没有一种方式，能把这个“自主迭代”的思想推广到任何有可量化指标的任务上呢？
我今天要分享的就是这样一个项目——AutoResearch，由我（Udit Goenka）开源的Claude Code Skill。它把Karpathy的核心思想泛化了，现在可以应用在：
提升代码测试覆盖率到95%
降低前端bundle体积
优化Lighthouse性能分数
提高网页可访问性/SEO得分
甚至改进邮件文案的打开率、内容质量打分…… 只要你能给出一个明确的目标 + 一个可自动运行的量化指标（metric），它就能让Claude自己开始无限迭代！
工作原理超级清晰：
你告诉它：我想干嘛？（Goal）
用什么命令来测效果？（Verify command，比如npm test --coverage）
是越高越好还是越低越好？
然后输入 /autoresearch，它就开始自主循环： 修改一点点代码/内容 → git commit → 跑verify命令测指标 → 变好了就保留，变差了自动git reset回滚 → 再来下一轮
